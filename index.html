Нейроотдел — интенсив по AI-агентам

Семь дней: от установки агента до команды нейросетей с общей памятью и монетизации. Каждый день — самостоятельный блок: понятная теория в двух фразах, затем готовые команды и промпты. Промпт копируешь в Claude, он ведёт тебя по шагам и подсказывает что делать. На ручных шагах (оплата, токены, регистрация) останавливаешься и делаешь сам.

Модели: Opus 4.8 — основная (код, ревью, решения). Sonnet 4.6 — субагенты (ресёрч, массовая обработка).

День 1 — Настройка Claude Code

вставь в Claude — он проведёт по шагам

Установка агента, базовые правила, первые скиллы, подключение GitHub. К концу дня — рабочий Claude Code, который понимает кто ты и действует по твоим правилам.

1. Что такое Claude Code

Claude Code — не чат. Это агент, который сам выполняет действия на компьютере: пишет код, запускает команды, создаёт и правит файлы, помнит контекст между сессиями. ChatGPT отвечает словами — Claude Code делает: ты говоришь «сделай», он делает. Работает в терминале или внутри редактора Cursor / VS Code.

Понадобится: macOS / Linux / Windows (WSL2), Node.js 22+, Git, подписка Anthropic (Pro 20$ или Max 100$).

2. Установка

Терминал — окно, где команды отдают текстом, а не мышкой. Открой его и выполни блок:

bash
npm install -g @anthropic-ai/claude-code
claude login
claude --version

claude login откроет вход через браузер. После авторизации claude --version покажет номер версии.

3. CLAUDE.md — паспорт агента

CLAUDE.md — файл, который агент читает в начале каждой сессии. В нём записано: кто ты, на каком языке отвечать, какой стек, что можно делать самому, а что только с твоего разрешения. Память агента живёт в этом файле, а не в чате: диалог стирается, CLAUDE.md остаётся. Без него агент — чистый лист.

Создай проект и сгенерируй стартовый файл командой /init:

bash
mkdir my-project && cd my-project && claude
/init

Затем доведи его под себя. Вставь в Claude промпт:

text
Собери CLAUDE.md для этого проекта. Сначала просканируй репозиторий, потом задай вопросы по очереди: кто я и язык общения; стек (язык, фреймворк, база, деплой); команды запуска и тестов; что тебе можно делать автономно, а что только с моего разрешения; правила git. После ответов сгенерируй файл по секциям Identity / Project / Workflow / Boundaries / Style, до 200 строк, и сохрани в ./CLAUDE.md.

4. GitHub — где живёт код

Репозиторий — папка проекта под версионным контролем. Git ведёт историю изменений (можно откатиться назад), GitHub хранит этот код в облаке. Файл .env с секретами в репозиторий не выкладывают никогда.

bash
gh auth login
gh repo create my-project --private --source=. --remote=origin --push

Или поручи агенту — вставь в Claude:

text
Подключи проект к GitHub. Проверь и при необходимости установи gh CLI, проведи меня через gh auth login. Собери .gitignore под мой стек и обязательно добавь .env, *.key, *.pem, secrets/. Сделай первый коммит и создай приватный репозиторий с пушем. После каждого шага говори что получилось.

5. Скиллы — готовые навыки агента

Скилл — готовая инструкция-навык, агент включает её сам по ключевым словам. Поставь три базовых.

СкиллЧто делает
SuperpowersМетодология: сначала план и тест, потом код
senior-brainstormАрхитектор: стек, дизайн системы
telegram-bot-builderСборка Telegram-ботов
bash
/plugin install superpowers@claude-plugins-official
claude skill add https://github.com/qwwiwi/agentos-skills/raw/main/senior-brainstorm.skill
claude skill add https://github.com/qwwiwi/agentos-skills/raw/main/telegram-bot-builder.skill

6. Автономный режим и экономия

Чтобы агент не спрашивал разрешения на каждое действие, есть три режима (переключение Shift+Tab): обычный, авто-правки, полностью автономный. Опасные команды (удаление, force push) остаются под запретом всегда. Настройки и список разрешённых команд — в ~/.claude/settings.json.

Контекст — память агента в рамках одной сессии. Чтобы тратить её экономно, поставь предел сжатия:

bash
export CLAUDE_CODE_AUTO_COMPACT_WINDOW=400000

День 2 — Проект с нуля

вставь в Claude — он проведёт по шагам

От пустой папки до рабочего приложения с тестовой и боевой площадкой. К концу дня — MVP, развёрнутый на staging, с пройденным аудитом безопасности.

1. Правила и репозиторий

Перед кодом агенту нужны три вещи: он сам, правила и репозиторий. Папка .claude хранит настройки агента внутри проекта (правила, скиллы) и едет вместе с кодом в git. CLAUDE.md держи коротким (до 150 строк), детали выноси в отдельные файлы правил.

Шаблон CLAUDE.md проекта:

markdown
# <Проект>
## Mission — что строим и для кого, одно предложение
## Stack — Next.js + TypeScript; Supabase (база, вход, хранилище); Tailwind; тесты Playwright
## Architecture — app/ (страницы); components/ (интерфейс); lib/ (серверная логика); tests/
## Commands — dev / lint / test / build
## Hard rules — без any в TypeScript; стили внутри компонента; без зелёных тестов не коммитим
## Environments — Local; Staging; Production
## Style — общение по-русски, код и комментарии по-английски, без emoji

2. Дизайн до кода

MVP — минимальная рабочая версия продукта. Прежде чем писать код, зафиксируй как продукт выглядит, в файле design.md: цвета и шрифты, компоненты, экраны, состояния (загрузка, пусто, ошибка). Агент берёт оттуда стиль и не выдумывает каждый раз заново.

markdown
# Design
## Tokens — цвета, шрифты (Inter + заголовочный), отступы, скругления
## Components — кнопка (варианты, размеры); карточка; поле ввода
## Screens — / (лендинг); /login; /app; /settings
## States — загрузка (скелетон); пусто (иллюстрация + кнопка); ошибка (текст + повтор)

3. Бэкенд под нагрузку

Бэкенд — то, что считает и хранит данные на сервере. Выбор зависит от числа пользователей.

Сколько пользователейЧто брать
10–100 (хобби, MVP)Обычный VPS + SQLite, дёшево
50 000+ (продукт, SaaS)Supabase или Yandex Cloud
Внутренний дашбордFirebase, бесплатно годами

Supabase — готовый бэкенд: база данных, вход пользователей, хранилище файлов из коробки. Поднимается за 15 минут, берём его. RLS в нём — правила доступа: каждый видит только свои данные.

4. Сборка одним промптом

У агента уже всё есть: CLAUDE.md, дизайн, выбранный бэкенд. Вставь в Claude:

text
Собери MVP по нашему CLAUDE.md и design.md. Проговори план, создай структуру по разделу Architecture, свёрстай компоненты по дизайну, подключи базу, запусти локально, поймай ошибки и почини. Покажи результат.

5. Площадки: тест и бой

Три окружения: Local (свой компьютер, ломаем спокойно), Staging — тестовая площадка с отдельной базой (проверяем), Production — боевая, где живые люди и деньги. Тесты гоняют против staging, никогда против боевой. Деплой — выкладка проекта на сервер.

text
Подготовь проект к staging и production. Сделай .env.example с пустыми заготовками всех ключей, добавь .env в .gitignore, вынеси адреса в переменные окружения. Опиши в README как поднять отдельный staging. Настрой GitHub Action: при изменении гонять тесты и выкладывать staging. Покажи изменения, не пушь.

6. Ревью и аудит безопасности

Перед боем прогоняем код через проверки по порядку: линтер (автопроверка стиля и ошибок), затем тесты, затем аудит безопасности. Вставь в Claude:

text
Проведи аудит безопасности фронта и бэка по семи областям: секреты в коде и .env; что утекает во фронтенд; проверка прав в каждом запросе; правила доступа в базе; подписи вебхуков; заголовки и CORS; уязвимые зависимости. Отчёт таблицей с уровнями Critical / High / Medium / Low. После отчёта не чини, жди подтверждения.

Закрытие найденного — отдельным промптом «закрой все critical и high», затем перепрогон до нуля критичных.

  • Репозиторий с CLAUDE.md, design.md и правилами
  • MVP на Supabase, работает локально
  • Staging с отдельной базой и автодеплоем
  • Зелёные линтер и тесты

День 3 — Общая память (Second Brain)

вставь в Claude — он проведёт по шагам

Один «второй мозг» на всю команду агентов: куда все пишут и откуда все читают. К концу дня — общая память на сервере, в которую попадают данные из всех каналов.

1. Зачем общая память

Память агента в чате стирается. Чтобы знания копились, нужен отдельный слой — Second Brain: база на сервере, где факты, решения и заметки хранятся постоянно и ищутся по смыслу. Один общий «мозг» для всех агентов лучше, чем у каждого своя записная книжка.

Память делится на слои: горячая (последнее, всегда под рукой), тёплая (решения, подгружается на старте), холодная (архив, по запросу).

2. Из чего состоит

gbrain — наша реализация второго мозга. Внутри: база с поиском по смыслу и несколько серверов-переходников (MCP), через которые агент пишет и читает.

СерверПортНазначение
memory8767запись заметок
recall8768поиск по смыслу
swarm8766обмен задачами между агентами
tasks8769задачи

MCP — протокол-переходник: через него агент подключает внешние инструменты (память, соцсети, задачи). Embedding — способ хранить смысл текста числами, чтобы искать по смыслу, а не по точному слову.

3. Развёртывание на сервере

VPS — арендованный сервер в облаке, работает круглосуточно. Поставь общую память под отдельным пользователем. Вставь в Claude:

text
Ты мой DevOps-агент. Разверни общую память (Second Brain) на этом VPS под отдельным пользователем. Создай пользователя, разверни систему из репозитория, подними серверы памяти, заведи первого агента для мониторинга и приёма входящих. Команды давай блоками, после каждого этапа показывай что получилось, в конце сделай сквозной тест. Токены и ключи не выводи в ответ.

4. Подключение агента к памяти

Агент читает свою память слоями через подключения в CLAUDE.md:

text
@core/USER.md
@core/rules.md
@core/warm/decisions.md
@core/hot/handoff.md

Каждый агент пишет в общую память по своим правам (RBAC): своя зона своя, чужое менять нельзя. Запись идёт сразу в двух местах — локальный файл и общая база (dual-write), чтобы ничего не терялось.

5. Источники наполнения

В общую память стекаются данные из всех каналов: Telegram, Instagram, YouTube, X, веб-страницы, голосовые, браузер. Каждый канал — свой скилл, который кладёт заметку в базу. Дальше всё ищется по смыслу за пару секунд.

Результат дня 3

  • Общая память развёрнута на сервере
  • Серверы памяти подняты, сквозной тест прошёл
  • Первый агент подключён, читает память слоями
  • Понятно как наполнять память из каналов

День 4 — Команда агентов и соцсети

вставь в Claude — он проведёт по шагам

Несколько агентов с ролями и общей памятью плюс навыки работы с соцсетями. К концу дня — команда (кодер, память, маркетолог) и подключённые каналы.

1. Три агента с ролями

Каждый агент — отдельная папка с своим CLAUDE.md и своим Telegram-ботом, но память общая. Заведём троих: кодер, агент памяти, маркетолог.

text
Подними команду из трёх агентов по инструкции AGENTIC-SWARM. Каждому поставь скилл Superpowers. Кодер — по шаблону coder, агент памяти — по шаблону inbox-monitoring, маркетолог — по шаблону marketer. Для каждого создай рабочую папку, скачай шаблон CLAUDE.md, установи скилл. Запусти каждого и убедись, что он представился своей ролью.

Боты создаются в @BotFather (официальный бот Telegram для создания ботов): три бота — три токена в файл секретов.

2. Как агенты общаются

Субагент — отдельный агент под одну подзадачу: главный раздаёт работу. Между собой агенты общаются через общую память (swarm), а не через ботов друг друга. При старте каждый агент проверяет, нет ли для него новых задач.

День 6 — Автономия агентов

вставь в Claude — он проведёт по шагам

Агенты работают сами по расписанию, не выходя за подписку. К концу дня — рабочие крон-задачи, хуки и обмен задачами между агентами.

1. Платное и бесплатное

С 15 июня 2026 запуск агента из скрипта (Agent SDK, claude -p, GitHub Actions) тарифицируется отдельно. А живая сессия, вызовы из неё, слэш-команды, хуки и субагенты остаются в подписке. Вывод: не дёргаем агента снаружи, а даём ему самому забирать задачи.

2. Агент сам забирает задачи

Pull-паттерн: расписание (крон) не вызывает агента, а только кладёт задачу в общую память. Живой агент сам периодически проверяет очередь и забирает. Крон — планировщик, запускает задачу по расписанию. Так всё остаётся в подписке.

text
Сделай крон-процесс. Раз в <период> он кладёт задачу в общую память (swarm). Агент в живой сессии сам заберёт её, проверяя очередь. Убедись, что крон не вызывает агента напрямую и не использует Agent SDK.

3. Хуки — действия без агента

Хук — скрипт, который срабатывает сам на событие (перед командой, после правки, в конце сессии). Работает помимо агента, поэтому он не может «забыть». Защитный хук (firewall) блокирует опасные команды до запуска:

bash
#!/bin/bash
set -euo pipefail
COMMAND="$1"
FORBIDDEN=("rm -rf /" "git push --force" "DROP TABLE")
for p in "${FORBIDDEN[@]}"; do
  if echo "$COMMAND" | grep -qE "$p"; then echo "BLOCKED: $p" >&2; exit 2; fi
done
exit 0

4. Сигнал «я жив» и обмен задачами

Heartbeat — сигнал «я жив» каждые несколько минут, заодно агент проверяет очередь задач. При старте каждый агент обязан: проверить входящие, взять новые задачи, отметиться живым. Так задача от одного агента доходит до другого.

Результат дня 6

  • Понятно что платно, а что в подписке
  • Крон кладёт задачи, агент сам их забирает
  • Стоит защитный хук на опасные команды
  • Агенты обмениваются задачами через общую память

Куда двигаться после интенсива и на чём строить продукт под масштаб. К концу дня — понимание моделей заработка и дешёвого продакшна.

1. Образовательный продукт: аудитория раньше платформы

Высокие чеки в 2026 продаются тяжело. Точка входа — недорогой мини-продукт под одну конкретную боль. Первый шаг — не платформа, а аудитория: рост охватов через короткие ролики (час в неделю с агентом на сценарии), затем подписчики, затем продукт. Тему берут из директа и пересылок, не из головы.

Воронка — путь клиента от первого касания до покупки. Лид-магнит — бесплатная польза за подписку. Трипваер — дешёвый первый продукт, снимает страх и ведёт к основному.

2. Агентство и процессы

У владельца две задачи: трафик и продукт. Агенты уменьшают штат, а не заменяют людей: где раньше нужна была команда, теперь хватает одного-двух человек с агентами. Все клиентские процессы (онбординг, КП, план, реализация, продление) живут в общей памяти, каждый этап — отдельный скилл.

4. Дешёвый продакшн под масштаб

Ротация подписок не масштабируется. Путь в продакшн на тысячи пользователей — недорогие модели (Minimax, DeepSeek) через API плюс свой Telegram-бот на своём сервере. Такие модели дешевле топовых в десятки раз и закрывают большинство задач, веб-поиск добавляют поверх (Grok, Gemini).
